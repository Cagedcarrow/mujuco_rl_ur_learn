# 在 Stable Baselines3 (SB3) 的 PPO 算法中，图像主要分为三大类：Rollout（数据采集）、Train（模型训练） 和 Time（时间效率）。

# 1. Rollout（数据采集层）：决定狗跑得好不好
# 这一块最直观，直接反应了机器狗在仿真环境里的表现。

# ep_rew_mean (回合平均奖励) ⭐最重要：
# 含义：机器狗在一次从"出生"到"摔倒/超时"过程中获得的总分平均值。
# 解读：必须持续上升。如果这条线在涨，说明狗学到了前进的技巧；如果一直横着走，说明它卡在了某个死胡同里（比如只会原地踏步）。

# ep_len_mean (回合平均长度)：
# 含义：机器狗在摔倒前能坚持多少步。
# 解读：如果你的奖励函数有"存活奖"，这条线会和奖励线同步上升。当它达到最大值（比如 1000 步）并拉成直线时，说明狗已经能稳稳地一直跑而不摔倒了。

# 2. Train（训练参数层）：决定脑子灵不灵
# 这一块反映了神经网络内部的数学更新是否健康。

# explained_variance (解释方差) ⭐核心指标：
# 含义：价值网络（Value Network）对奖励的预测准确度。
# 解读：越接近 1.0 越好。
# < 0：模型完全在瞎猜，步态会非常混乱。
# 0.5 ~ 0.8：模型已经摸透了环境规律。
# 0.9：非常完美的收敛状态。

# entropy_loss (策略熵损失)：
# 含义：代表动作的"随机性"或"好奇心"。
# 解读：应该缓慢下降。
# 初期：值很大（负数，如 -17），说明狗在尝试各种奇怪的姿势。
# 后期：值变小，说明狗已经形成了个性化的固定步态（收敛）。如果它下降太快，说明狗太早"固执己见"，可能没学到最优解。

# value_loss (价值函数损失)：
# 含义：预测的分数和实际得分的差距。
# 解读：通常会先升后降。初期它在探索高分，误差会变大；后期随着预测越来越准，这个值会降低。

# approx_kl (平均 KL 散度)：
# 含义：新策略和旧策略之间的差异。
# 解读：如果这个值突然飙升，说明训练崩了（神经网络更新幅度过大，狗突然变傻了）。PPO 算法会自动限制它在很小的范围内。

# 3. Time（时间效率层）：决定跑得快不快
# fps (每秒处理帧数)：
# 解读：这就是你 RTX 4090 D 的力量体现。1000+ 是极品速度。如果这个值在训练过程中大幅下降，说明你的 CPU 出现了发热降频或内存瓶颈。

# 4. 总结：什么样的图表是"完美的"？
# Reward (奖励)：像爬山一样稳定上升。
# Explained Variance (解释方差)：从 0 附近逐渐爬升到 0.8 以上。
# Entropy (熵)：平滑下降，但不归零（归零意味着完全丧失探索能力）。
# Learning Rate (学习率)：如果你用了衰减策略，它会往下走；如果你用的是固定的（如 3e-4），它就是平的。
